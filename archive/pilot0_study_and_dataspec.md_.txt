TELOS Pilot 0 Study & Data Specification

Runtime Governance Validation Protocol (Internal Pilot Study)

⸻

🧩 Overview

Purpose:
Pilot 0 is the first full-system test of TELOS as a runtime governance substrate for LLM sessions.
Its goal is to verify that the mathematical instrumentation functions correctly and that exported telemetry can demonstrate measurable drift, fidelity, and intervention dynamics across four governance conditions.

This is not a public-facing benchmark; it is an internal validation cycle to generate the data funders and reviewers will require.

⸻

🎯 Research Hypothesis & Objectives

Hypothesis:
Mathematically adaptive governance (TELOS runtime) yields higher stability and lower intervention burden than static or ungoverned modes.

Objectives
	1.	Verify end-to-end function of Primacy Attractor → Intervention Controller → Telemetry Export.
	2.	Compare drift and fidelity across four modes:
 A Stateless | B Prompt-Only | C Cadence | D TELOS Runtime
	3.	Capture timestamped telemetry for latency and user-burden analysis.
	4.	Produce reproducible JSON + CSV data suitable for correlation and validation studies.

⸻

⚙️ Study Conditions

Condition	Description	Intervention Mode
A Stateless	LLM runs without PAP (initial message only).	None
B Prompt-Only	Session begins with PAP declaration; no runtime monitoring.	None
C Cadence	PAP initialized; reminder prompt every 3 turns.	Fixed-interval
D TELOS Runtime	Continuous Primacy Attractor tracking; adaptive corrections when ε-threshold exceeded.	Proportional

Each scripted conversation is replayed under all 4 conditions with identical user input to allow direct comparison.

⸻

🧮 Instrumentation

Module	Function	Key Metrics
PrimacyAttractorMath	τ-weighted center & r = 2 / ρ basin geometry	Error signal (E), Lyapunov V(x), fidelity (F)
InterventionController	Monitors E and V(x); triggers reminder / regen / antimeta	Intervention count & type
UnifiedSteward	Session orchestration & logging	Per-turn telemetry JSON
Telemetry Exporter	Writes turn + session files	CSV + summary JSON
Observation Mode	Logs drift metrics only (no intervention)	Raw trajectory data


⸻

🧠 Execution Protocol

1 — Observation Mode Baseline

Run each scenario with interventions disabled to generate pure drift curves.

python -m telos_purpose.runners.TELOS_runner_script --mode observation

2 — Four-Condition Runs

python -m telos_purpose.validation.run_validation --study pilot_0

	•	Executes each conversation under all four modes.
	•	Exports telemetry per turn and summary per session.

3 — Forked Replays (Optional)

When drift > ε_min without correction, fork from that turn and resume governed run to compare trajectories.

🧾 Data Export Schema

A. Turn-Level Telemetry (CSV)

Stored as: /validation_results/{condition}/{session_id}_turns.csv

Field	Type	Description
session_id	str	Unique ID for run
condition	str	stateless / prompt_only / cadence / telos_governed
turn_id	int	Sequential turn
timestamp	ISO 8601	Time of generation
Δt_ms	int	Elapsed since previous turn
user_input	text	Raw user message
model_output	text	Model response
embedding_distance	float	Distance to attractor center
fidelity_score	float	Hard fidelity (0–1)
soft_fidelity	float	Soft fidelity (0–1)
lyapunov_delta	float	ΔV from previous turn
intervention_triggered	bool	True if any intervention fired
intervention_type	text	none
governance_drift_flag	bool	True if E > ε_min or outside basin
governance_correction_applied	bool	True if runtime applied fix
notes	text	Optional annotation
