## What is TELOS Observatory?

TELOS Observatory is the first **mathematically governed and security-validated** AI conversation platform. Watch your conversations stay on purpose in real-time, measure governance fidelity turn-by-turn, and export cryptographically signed evidence for research or compliance.

**The Problem:**
- LLMs drift from their purpose across conversations (20-40% governance loss)
- Privacy leakage in multi-turn sessions
- No verifiable audit trail for regulated domains

**The Solution:**
- Mathematical primacy attractors + real-time intervention
- Cryptographically signed telemetry (quantum-resistant)
- Zero PII leakage (validated against 2,842 attacks)
- Healthcare safety (validated against 1,586 medical harm scenarios)

## Mathematical Foundation

### Dual Primacy Attractor Architecture

**User PA** (Topic Governance):
```
â_user = (τ·p + (1-τ)·s) / ‖τ·p + (1-τ)·s‖

Where:
- p = purpose embedding
- s = scope embedding
- τ = task_priority parameter
```

**AI PA** (Behavioral Governance):
```
â_ai = Lock-On Derivation from User PA
     = Behavioral constraints embedding
```

**Privacy PA** (PII Protection - Deployment Agnostic):
```
â_privacy = Regulatory compliance embedding
          = {GDPR, CCPA, HIPAA, NIST 800-53, ISO 27701}
```

**Fidelity Measurement**:
```
F_user(x) = 1 if ‖x - â_user‖ < r_user, else 0
F_ai(x)   = 1 if ‖x - â_ai‖ < r_ai, else 0
F_privacy(x) = 1 if no PII detected in x, else 0

Overall Fidelity = F_user ∧ F_ai ∧ F_privacy
```

**Basin Radius**:
```
r = 2/max(ρ, 0.25)
ρ = 1 - constraint_tolerance
```

### Telemetric Keys (Quantum-Resistant Cryptography)

**Key Generation**:
```
K_session = SHA3-512(
    governance_telemetry ||
    session_id ||
    timestamp ||
    8_entropy_sources
)
```

**Signature**:
```
Sig(delta) = HMAC-SHA512(K_session, delta)

Where:
- delta = conversation state change (not full content)
- K_session = session-bound ephemeral key
- Destroyed post-session (forward secrecy)
```

**Security Properties**:
- 256-bit post-quantum security (Grover-resistant)
- No conversation content exposure (delta-only)
- Forward secrecy (key recovery impossible)
- Session binding (replay attacks prevented)

See `/docs/mathematical_foundations.md` and `/docs/telemetric_keys_specification.md` for complete derivations.
