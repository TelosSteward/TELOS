# TELOS Project Rules for Claude

## 🚨 CRITICAL REPOSITORY RULES 🚨

### Rule #1: NEVER Commit from telos_privacy to TELOS Repo
**NEVER, EVER commit files from `~/Desktop/telos_privacy/` to `github.com/TelosSteward/TELOS`**

**Why:**
- telos_privacy is the MESSY WORKSHOP
- TELOS repo is the CLEAN SHOWROOM
- Planning docs, WIP files, personal notes NEVER go to GitHub

### Rule #2: NO Planning Files in TELOS Repo
**ZERO planning files, notes, or work-in-progress documents ever committed to TELOS repo**

**Examples of what NEVER goes to GitHub:**
- ❌ Planning documents (*_PLAN.md, *_TODO.md)
- ❌ Session notes (SESSION_*.md, NEXT_SESSION_*.md)
- ❌ Work-in-progress features
- ❌ Test results (tests/validation_results/*)
- ❌ Personal notes
- ❌ Anything from telos_privacy/

### Rule #3: Work in Progress - KEEP LOCAL
**Live demo development and ANY work in progress:**
- ✅ Build in telos_privacy/
- ✅ Test locally
- ✅ Get user approval
- ✅ ONLY push to GitHub when FINISHED and approved

### Rule #4: Update Flow (The ONLY Way)

**When pushing updates to TELOS repo:**

1. **Work in**: `~/Desktop/telos_privacy/` (local only)
2. **When FINISHED**: Copy to `~/Desktop/Privacy_PreCommit/`
3. **Test it works** in Privacy_PreCommit
4. **Get user approval**: "Ready to push to GitHub?"
5. **THEN commit**: Only from Privacy_PreCommit
6. **Push**: `cd Privacy_PreCommit && git push origin main`

**NEVER skip steps 2-5!**
