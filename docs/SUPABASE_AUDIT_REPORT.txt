================================================================================

MAJOR GAPS:

1. MISSING: Turn Cache / Session State Persistence
   ├─ Problem: turn_cache is built dynamically but never stored to Supabase
   ├─ Location: Exists in st.session_state in beta_response_manager.py
   ├─ Should Save: 
   │   - Turn embeddings (user_embedding, response_embedding)
   │   - User fidelity (normalized and raw)
   │   - AI fidelity metrics
   │   - Adaptive context results
   │   - Intervention details
   │   - Response comparisons (native vs telos)
   ├─ Frequency: Every turn
   └─ Impact: Can't reconstruct full governance context from Supabase alone

2. MISSING: Governance Trace Events (Evidence Schema)
   ├─ Problem: Evidence schema exists (evidence_schema.py) but not integrated
   │           with backend client
   ├─ Event Types NOT saved to Supabase:
   │   - FidelityCalculatedEvent (raw similarity, normalized fidelity, zone)
   │   - InterventionTriggeredEvent (reason, strength, semantic band)
   │   - ResponseGeneratedEvent (source, length, generation time)
   │   - SSETokenEvent (streaming token tracking)
   │   - TurnCompleteEvent (comprehensive turn summary)
   ├─ Location: governance_trace_collector.py logs to JSONL locally
   │           but has NO connection to backend_client
   ├─ Missing: transmit_governance_events() method in BackendService
   └─ Impact: Governance trace data only available locally

3. MISSING: Intervention Semantic Details
   ├─ Problem: Semantic interpreter results not saved
   ├─ Should Save:
   │   - Semantic band (minimal, light, moderate, firm, strong)
   │   - Behavioral specifications (no hedging, use directives, etc.)
   │   - Context injection prompts used
   │   - Steward interpretation reasoning
   ├─ Location: computed in beta_response_manager.py but stored locally only
   └─ Impact: Can't analyze intervention effectiveness patterns

4. MISSING: Adaptive Context Decisions
   ├─ Problem: Adaptive context manager processes every message
   ├─ Should Save:
   │   - Message type classification (DIRECT, FOLLOW_UP, CLARIFICATION, ANAPHORA)

   │   - Context buffer state

   └─ Impact: Lost valuable context patterns for analysis

5. MISSING: AI PA Fidelity & Response Analysis
   ├─ Problem: AI response fidelity calculated but not stored systematically
   ├─ Should Save:
   │   - AI response raw similarity (before normalization)
   │   - AI response normalized fidelity
   │   - Intervention applied to AI response (yes/no)
   │   - Regeneration attempts and outcomes
   │   - Response realignment metrics
   ├─ Location: computed in beta_response_manager.py line 542-549+

   └─ Impact: Can't measure AI response quality independently

6. MISSING: Semantic Continuity Inheritance (SCI) Data

   ├─ Should Save:
   │   - Previous turn context match score
   │   - Semantic inheritance applied (yes/no)

   ├─ Location: adaptive_context.py, not persisted
   └─ Impact: Can't validate SCI effectiveness

7. MISSING: Failure & Error Events

  - Regular privacy audit of stored field values
