## Detailed Competitor Analysis

### 1. Microsoft Azure Prompt Shields

**What They Do:**
- Detect jailbreak attempts (direct prompt attacks)
- Detect indirect attacks (embedded in documents)
- Binary classification: attack detected / not detected
- ML classifiers trained on threat patterns

**Limitations:**
- **Wrong question**: Asks "is this harmful?" not "does this serve purpose?"
- **Binary only**: No graduated response, no proportional intervention
- **No trajectory**: Each request evaluated in isolation
- **No purpose concept**: Cannot measure alignment to user intent

**TELOS Advantage:**
Microsoft catches malicious prompts. TELOS catches purpose drift—a well-meaning but wandering agent that gradually moves away from user intent without triggering any harm filters.

---

### 2. Arthur AI (Agent Discovery & Governance)

**What They Do (December 2025 launch):**
- Automated discovery and cataloging of agents across environments
- Monitoring of commercial agentic AI solutions
- Acceptable use and security policies
- Data protection (PII, PHI, IP safeguards)
- Threshold-based alerts

**Positioning:** "Turn agent chaos into structured, scalable operation"

**Limitations:**
- **Policy-based**: Rules defined by humans, not mathematical measurement
- **Observability focus**: Watches agents, doesn't mathematically govern them
- **No fidelity concept**: Cannot measure continuous alignment to purpose
- **Threshold alerts**: Binary alerting, not graduated intervention

**TELOS Advantage:**
Arthur tells you *what agents you have* and *if they violate policies*. TELOS measures *whether each agent action aligns with declared purpose* and intervenes proportionally. Arthur is the map; TELOS is the steering system.

---

### 3. Patronus AI

**What They Do:**
- Agent debugging (Percival)
- Evaluation benchmarks (TRAIL)
- Generative simulators for testing
- LLM-as-judge evaluation

**Limitations:**
- **Post-hoc only**: Evaluates after the fact, doesn't govern in real-time
- **Testing, not governance**: Tells you what went wrong, doesn't prevent it
- **LLM judges**: Subjective evaluation, not mathematical measurement
- **No intervention capability**: Cannot redirect agent behavior

**TELOS Advantage:**
Patronus tests agents in development. TELOS governs agents in production. You need both—but only TELOS provides runtime protection.
