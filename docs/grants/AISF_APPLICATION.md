- **Purpose**: Human definition of intent, objectives, constraints
- **Authority**: Ultimate control exercised continuously (not as escalation)

**Primacy Attractor = Instantiated Constitutional Law for Session-Level Governance**

**How Constitutional Law is Instantiated**:
1. **Human-Authored**: Constitutional actors declare constraints for the session
2. **Instantiated**: Declared constraints become enforceable constitutional requirements
3. **Ephemeral**: Constitutional law exists within session state (not permanent model modification)
4. **Enforced**: The Constitutional Filter enforces this law through orchestration layer in real-time

### Three-Tier Governor Architecture

**Tier 3: Human Authority (Primacy)**
- Domain experts exercise ultimate control
- **Author constitutional requirements** for session-level governance (privacy + purpose)
- Review escalated edge cases
- **Not exception handling—this is authoring the law AI must follow**

**Tier 2: Constitutional Filter (Governance Infrastructure)**
- Complete governance system (not component or layer)
- **Primacy Attractor = instantiated constitutional law** for the session

**Tier 1: AI Subsystem (Constrained Operation)**
- LLM operates within primacy-defined bounds
- Generates responses under governance constraints
- Escalates when approaching boundaries
- **Functions as tool under authority, not autonomous agent**

### Information Flow
- **Upward**: Violations escalate to authority (Tier 1 → 2 → 3)
- **Downward**: Boundaries constrain operation (Tier 3 → 2 → 1)

**Key Principle**: Humans are not "fixing AI mistakes"—they're exercising governance. AI is not "in control"—it operates under architectural constraints.
